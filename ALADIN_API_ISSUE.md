# 알라딘 Open API 서버 사이드 호출 제한 문제

## 문제 상황

알라딘 Open API를 서버 사이드(Next.js API Route)에서 호출할 때 "API출력이 금지된 회원입니다" 오류가 발생합니다.

### 원인
- 알라딘 Open API는 등록된 도메인에서의 요청만 허용
- 도메인 확인을 위해 `Referer` 헤더를 사용
- 서버 사이드 호출 시 Referer 헤더가 제대로 전달되지 않거나 무시될 수 있음
- 알라딘 API가 서버 간 호출을 차단할 수 있음

## 해결 방안

### 방안 1: 알라딘 고객센터 문의 (권장)
알라딘 Open API 고객센터에 다음 사항을 문의:
- 서버 사이드(백엔드)에서 API 호출이 가능한지
- 서버 사이드 호출 시 도메인 인증 방법
- IP 화이트리스트 등 추가 인증 방법

### 방안 2: 클라이언트 사이드 직접 호출
- 브라우저에서 직접 알라딘 API 호출
- **주의**: API 키가 노출되므로 프로덕션 환경에서는 권장하지 않음
- CORS 문제 발생 가능

### 방안 3: 대체 API 사용
- **도서관 정보나루 API**: 무료, 서버 사이드 호출 지원
- **네이버 검색 API**: 도서 검색 지원
- **Google Books API**: 무료, 제한적

### 방안 4: 프록시 서버 사용
- 중간 프록시 서버를 통해 호출
- 복잡도 증가

## 현재 상태
- 등록된 URL: `http://reading-tree-project.vercel.app`
- API 키: `ttbkhami10002233001`
- 문제: 서버 사이드 호출 시 도메인 인증 실패
