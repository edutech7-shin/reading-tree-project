### ✅ 3. 핵심 보상 시스템 (MVP)

| 기능 | 상태 | 구현 위치 | 비고 |
|------|------|-----------|------|
| 학생 기록 제출 | ✅ 완료 | `app/record/page.tsx` | `status='pending'`으로 저장 |
| 교사 승인 처리 | ✅ 완료 | `app/teacher/approve/page.tsx` | 승인 버튼 클릭 시 `approve_record_and_reward()` 호출 |
| 자동 보상 실행 (잎사귀 +1) | ✅ 완료 | `supabase/migrations/20251031_init.sql` | `approve_record_and_reward()` 함수 구현 |
| 자동 보상 실행 (포인트 +10) | ✅ 완료 | 동일 | `profiles.points` 자동 업데이트 |
| 자동 보상 실행 (레벨업 체크) | ✅ 완료 | 동일 | `current_leaves >= level_up_target` 시 자동 레벨업 |
| 나무 성장 (레벨업) | ✅ 완료 | 동일 | 레벨업 시 `current_level +1`, `current_leaves = 0` |

## 🚨 우선적으로 구현해야 할 기능

### 1. 높은 우선순위

1. **우리 반 나무 데이터 연동** ⚠️
   - 현재: `app/page.tsx`에서 하드코딩된 값 사용
   - 필요: 실제 `class_trees` 테이블에서 데이터 가져오기

2. **학생별 잎사귀 수 표시** ⚠️
   - 현재: 학생 관리 화면에 레벨과 포인트만 표시
   - 필요: 각 학생의 승인된 독서 기록 수 표시

3. **내 나무 - 독서 목록** ❌
   - 현재: 개수만 표시
   - 필요: 내가 기록한 독서 목록을 리스트로 표시

### 2. 중간 우선순위

4. **교사 코멘트 기능** ❌
   - 반려 시 코멘트 입력 필드 추가

5. **반 나무 목표치 설정** ❌
   - 교사가 `level_up_target`을 수정할 수 있는 UI

### 3. 낮은 우선순위 (MVP 범위 밖일 수 있음)

6. **책 검색 API 연동** ❌
   - 알라딘 또는 도서관 정보나루 API 연동
   - `book_cache` 테이블 활용

7. **학생 계정 생성 UI** ⚠️
   - 교사가 직접 학생 계정을 생성할 수 있는 화면
   - 현재는 회원가입 페이지만 있음
